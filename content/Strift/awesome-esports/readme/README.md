# Awesome Esports [![Awesome](https://awesome.re/badge.svg)](https://awesome.re)

> A curated list of open-source projects related to esports.

[Esports](https://en.wikipedia.org/wiki/Esports) (also known as electronic sports or e-sports) is a form of competition using video games. It often takes the form of organized, multiplayer video game competitions, particularly between professional players, individually or as teams.

Unmaintained projects, now only relevant for educational purposes, are in the [Archive](https://github.com/Strift/awesome-esports/blob/main/README.md/ARCHIVE.md).

## Libraries

> Programming libraries organized per use cases.

### APIs & Web Services

*   [Blizzard.js (‚≠ê139)](https://github.com/benweier/blizzard.js) - A Node.js library for interacting with the Blizzard Community Platform API.
*   [Discord.js (‚≠ê24k)](https://github.com/discordjs/discord.js) - A Node.js library for interacting with the Discord API.
*   [HLTV (‚≠ê341)](https://github.com/gigobyte/HLTV) - A Node.js library for interacting with the HLTV API.
*   [liquipediapy (‚≠ê61)](https://github.com/c00kie17/liquipediapy) - A Python library for interacting with the Liquipedia API.
*   [steam (‚≠ê911)](https://github.com/ValvePython/steam/) - A Python library for interacting with various parts of Steam.
*   [Steam Community (‚≠ê418)](https://github.com/DoctorMcKay/node-steamcommunity) - A Node.js library for interacting with the Steam Community website.
*   SteamID - A library for manipulating Steam IDs.
    *   [Node.js (‚≠ê65)](https://github.com/DoctorMcKay/node-steamid)
    *   [PHP (‚≠ê8)](https://github.com/DoctorMcKay/php-steamid)
*   [SteamUser (‚≠ê719)](https://github.com/DoctorMcKay/node-steam-user) - A Node.js library for interacting with the Steam network via the Steam client protocol.
*   [Steam TOTP (‚≠ê217)](https://github.com/DoctorMcKay/node-steam-totp) - A Node.js library for generating Steam-style 2FA codes.
*   [valorant.js (‚≠ê98)](https://github.com/liamcottle/valorant.js) - A Node.js library for interacting with the VALORANT APIs used in game.

### Authentication

*   [Passport-Steam (‚≠ê325)](https://github.com/liamcurry/passport-steam) - A Node.js passport authentication strategy for Steam.

### Data parsing and analysis

*   [awpy (‚≠ê312)](https://github.com/pnxenopoulos/awpy) - A Python library for parsing, analyzing, and visualizing CS:GO data.
*   [Boxcars (‚≠ê81)](https://github.com/nickbabcock/boxcars) - A Rust library for parsing Rocket League replays.
*   [Cassiopeia (‚≠ê516)](https://github.com/meraki-analytics/cassiopeia) - A Python framework for interacting with and analyzing data from the Riot Games League of Legends API.
*   [Clarity (‚≠ê581)](https://github.com/skadistats/clarity) - A Java library for parsing CS:GO and Dota 2 replays.
*   [demoinfocs-golang (‚≠ê545)](https://github.com/markus-wa/demoinfocs-golang) - A Go library for parsing and analyzing CS:GO demos (ie. replays.)
*   [Rattletrap (‚≠ê140)](https://github.com/tfausak/rattletrap) - A Haskell library for parsing and generating Rocket League replays.

### Team management

*   [LoL in-house bot (‚≠ê110)](https://github.com/mrtolkien/inhouse_bot) - A Discord bot handling role queue, matchmaking, and rankings for League of Legends in-house games.

### Tournaments management

*   [brackets-manager.js (‚≠ê172)](https://github.com/Drarig29/brackets-manager.js) - A JavaScript library to manage tournament brackets.
*   [brackets-viewer.js (‚≠ê126)](https://github.com/Drarig29/brackets-viewer.js) - A JavaScript library to display tournament brackets.
