# Sublime Text Bookmarks [![Awesome](https://cdn.rawgit.com/sindresorhus/awesome/d7305f38d29fed78fa85652e3a63e154dd8e8829/media/badge.svg)](https://github.com/sindresorhus/awesome)

> [Sublime Text](http://www.sublimetext.com/) is a cross-platform text and source code editor, with a Python application programming interface. Its functionality is extendable with plugins. Most of the extending packages have free-software licenses and are community-built and maintained. ‚Äî [Wikipedia](https://en.wikipedia.org/wiki/Sublime_Text)

This tiny project follows [GitHub community trend (‚≠ê223k)](https://github.com/sindresorhus/awesome) to aggregate the most essential bookmarks for specific subject in the form of a handy well-structured collection. Here you will find tutorials and learning materials for Sublime Text, general purpose extensions for coding and text editing, and specialized extensions grouped by usage profiles.

This list not supposed to include absolutely all Sublime Text plugins, due we already have [Package Control](https://packagecontrol.io/) for this. It is intended to be a starting point helping to setup working environment, or check out for new extensions to make your existing Sublime setup more awesome. You may find ‚òÖ here and there in the list. It stands for Editors' Choice. Sharing and pull requests are very much appreciated!

**[Fork and edit (‚≠ê948)](https://github.com/dreikanter/sublime-bookmarks/edit/master/README.md)** (or [propose something (‚≠ê948)](https://github.com/dreikanter/sublime-bookmarks/issues/new) to add).

## Extensions

Alive projects only, with Sublime Text version 3 support.

### General Purpose

*   [LSP](https://packagecontrol.io/packages/LSP) ‚Äî Language Server Protocol support for Sublime Text 3 that gives you IDE features.
*   [Sidebar Enhancements (‚≠ê2.1k)](https://github.com/titoBouzout/SideBarEnhancements) ‚Äî provides enhancements to the operations on Sidebar of Files and Folders.
*   [Trailing Spaces (‚≠ê884)](https://github.com/SublimeText/TrailingSpaces) ‚Äî highlight trailing spaces and delete them in a flash.
*   [GitGutter (‚≠ê3.9k)](https://github.com/jisaacks/GitGutter) ‚Äî a plugin to see git diff in gutter.
*   [EditorConfig (‚≠ê1.8k)](https://github.com/sindresorhus/editorconfig-sublime) ‚Äî .editorconfig support.
*   [SublimeLinter (‚≠ê1.9k)](https://github.com/SublimeLinter/SublimeLinter3) ‚Äî a plugin that provides a framework for linting code. Whatever language you code in, SublimeLinter can help you write cleaner, better, more bug-free code. SublimeLinter has been designed to provide maximum flexibility and usability for users and maximum simplicity for linter authors.
*   [Sublime Worksheet (‚≠ê334)](https://github.com/jcartledge/sublime-worksheet) ‚Äî an inline REPL for JavaScript, PHP, Ruby and more.
*   [Sublime Alignment (‚≠ê514)](https://github.com/wbond/sublime_alignment) ‚Äî a simple key-binding for aligning multi-line and multiple selections.
*   [ApplySyntax (‚≠ê336)](https://github.com/facelessuser/ApplySyntax) ‚Äî a plugin that allows to detect and apply the syntax of files that might not otherwise be detected properly. For example, files with the .rb extension are usually Ruby files, but when they are found in a Rails project, they could be RSpec spec files, Cucumber step files, Ruby on Rails files (controllers, models, etc), or just plain Ruby files.
*   [All Autocomplete (‚≠ê907)](https://github.com/alienhard/SublimeAllAutocomplete) ‚Äî extends the default autocomplete to find matches in all open files.
*   [SublimeREPL (‚≠ê2.1k)](https://github.com/wuub/SublimeREPL) ‚Äî runs an interpreter inside ST2 (Clojure, CoffeeScript, F#, Groovy, Haskell, Lua, MozRepl, NodeJS, Python, R, Ruby, Scala, shell or configure one yourself).
*   [FindKeyConflicts (‚≠ê145)](https://github.com/skuroda/FindKeyConflicts) ‚Äî identify conflicting key mappings.
*   [Glue (‚≠ê255)](https://github.com/chrissimpkins/glue) ‚Äî a cross-platform, extensible plug-in for Sublime Text 2 and 3 that connects your favorite editor to your shell.
*   [File Navigator (‚≠ê55)](https://github.com/csch0/SublimeText-File-Navigator) ‚Äî keyboard-friendly file navigator and manager.
*   [Synesthesia (‚≠ê17)](https://github.com/dariusf/synesthesia) ‚Äî a plugin for highlighting text with pretty colours.
*   [StyleToken (‚≠ê47)](https://github.com/vcharnahrebel/style-token) ‚Äî allows to highlight certain pieces of text with different colors (similar to Notepad++ "Style token" functionality).
*   [Origami (‚≠ê1.2k)](https://github.com/SublimeText/Origami) ‚Äî split the window however you like! Create new panes, delete panes, move and clone views from pane to pane.
*   [Sync Settings (‚≠ê279)](https://github.com/mfuentesg/SyncSettings) - The cross-platform solution to keep Sublime Text configuration synchronized
*   [SublimeFileBrowser (‚≠ê388)](https://github.com/aziz/SublimeFileBrowser) - Ditch the sidebar and browse your files in a normal tab with your keyboard, like a pro! Also, [Dired](https://en.wikipedia.org/wiki/Dired) lets you modify your directory structure using the powerful text editing feature that ST provides.
*   [PackageResourceViewer (‚≠ê289)](https://github.com/skuroda/PackageResourceViewer) ‚Äî View and Edit Sublime package resources. Helpful for extensions and themes development.
