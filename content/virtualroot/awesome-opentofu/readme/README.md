## Tools

### Environment managers

*   [arkade (‚≠ê4.4k)](https://github.com/alexellis/arkade) - CLI and Kubernetes app installer.
*   [asdf-opentofu (‚≠ê28)](https://github.com/virtualroot/asdf-opentofu) - OpenTofu plugin for asdf version manager.
*   [tenv (‚≠ê1k)](https://github.com/tofuutils/tenv) - Terraform and OpenTofu version manager written in Go.
*   [tfswitcher (‚≠ê4)](https://github.com/ASleepyCat/tfswitcher) - Terraform and OpenTofu version switcher written in Rust.
*   [tofuenv (‚≠ê181)](https://github.com/tofuutils/tofuenv) - OpenTofu version manager inspired by tfenv.

### Wrappers

*Simplify your OpenTofu workflows with a thin wrapper.*

*   [Atmos (‚≠ê953)](https://github.com/cloudposse/atmos) - Orchestration tool that keeps environment configuration DRY.
*   [Terragrunt (‚≠ê8.6k)](https://github.com/gruntwork-io/terragrunt) - Keep your configurations DRY, work with multiple modules, and manage remote state.
*   [Terramate (‚≠ê3.4k)](https://github.com/terramate-io/terramate) - Automation, orchestration, and code generation for OpenTofu, Terraform, Kubernetes, and others.
*   [easy\_infra (‚≠ê72)](https://github.com/SeisoLLC/easy_infra) - Docker container to simplify and secure the use of infrastructure as code.
*   [pug (‚≠ê534)](https://github.com/leg100/pug) - A terminal user interface for power users.
*   [tf (‚≠ê78)](https://github.com/dex4er/tf) - Less verbose and more friendly command outputs.
*   [tfam (‚≠ê23)](https://github.com/Ant0wan/tfam) - Rust-powered wrapper for concurrent Terraform/OpenTofu apply, enabling multi-deployment support.
*   [tfexe (‚≠ê3)](https://github.com/Ant0wan/tfexe) - Rust-powered wrapper for seamless execution of tfswitch and Terraform/OpenTofu with version control.
*   [tfwrapper (‚≠ê142)](https://github.com/claranet/tfwrapper) - Python wrapper that simplifies OpenTofu usage and enforces best practices.

### CI

*   [Atlantis](https://www.runatlantis.io/) - Automating workflows via pull requests.
*   [TF-via-PR (‚≠ê196)](https://github.com/OP5dev/TF-via-PR) - GitHub Action to init, plan and apply Terraform/OpenTofu via PR automation.
*   [pre-commit-opentofu (‚≠ê56)](https://github.com/tofuutils/pre-commit-opentofu) - Git pre-commit hooks plugin.
*   [setup-opentofu (‚≠ê101)](https://github.com/opentofu/setup-opentofu) - Set up OpenTofu CLI in your GitHub Actions workflow.
*   [terraform-github-actions (‚≠ê836)](https://github.com/dflook/terraform-github-actions) - GitHub Actions for OpenTofu.
*   [tofu-controller (‚≠ê1.4k)](https://github.com/flux-iac/tofu-controller) - GitOps OpenTofu and Terraform controller for Flux.

### Tests

*   [Terratest (‚≠ê7.6k)](https://github.com/gruntwork-io/terratest) - Go library that makes writing automated tests for your infrastructure code easier.

### State

*Analyze and manipulate OpenTofu's state.*

*   [tfmigrate (‚≠ê1.2k)](https://github.com/minamijoyo/tfmigrate) - State migration tool.

### Providers

*Inspect and interact with OpenTofu providers.*

*   [tfschema (‚≠ê299)](https://github.com/minamijoyo/tfschema) - Schema inspector for providers.

### Platforms

*Alternatives to Terraform Cloud.*

*   [digger (‚≠ê4.5k)](https://github.com/diggerhq/digger) - Open-source IaC orchestration tool. Digger allows you to run IaC in your existing CI pipeline.
*   [terrakube (‚≠ê626)](https://github.com/AzBuilder/terrakube) - Open-source platform with a private registry, remote state, custom flows, scheduled workspaces, and visual states.
*   [tofutf (‚≠ê77)](https://github.com/tofutf/tofutf) - Open-source alternative to Terraform Enterprise with SSO, team management, agents, etc.
*   [Terrateam](https://terrateam.io) - Open-source alternative to Terraform Cloud/Enterprise, GitOps-first with native GitHub integration and designed for scale, security, and reliability.

### Registry

*   [library.tf](https://library.tf/) - An indexer of registries for providers and modules with insights and documentation.
*   [boring-registry (‚≠ê202)](https://github.com/boring-registry/boring-registry) - Boring-registry is an open-source module and provider registry compatible with OpenTofu.
*   [hermitcrab (‚≠ê29)](https://github.com/seal-io/hermitcrab) - Registry network mirroring service compatible with OpenTofu.
*   [terrac (‚≠ê31)](https://github.com/haoliangyu/terrac) - Minimal private module registry compatible with OpenTofu.
*   [GitLab Module Registry](https://docs.gitlab.com/ee/user/packages/terraform_module_registry/) - Use GitLab projects as a private registry for terraform modules.
*   [terralist (‚≠ê385)](https://github.com/terralist/terralist) - Private registry for providers and modules.
*   [citizen (‚≠ê639)](https://github.com/outsideris/citizen) - Private registry for modules and providers with support for multiple databases and storages.
*   [petra (‚≠ê44)](https://github.com/devoteamgcloud/petra) - Private registry manager using Google Cloud Storage.
*   [tapir (‚≠ê218)](https://github.com/PacoVK/tapir) - Private registry for modules and providers with a UI.
*   [terraform-registry (‚≠ê105)](https://github.com/nrkno/terraform-registry) - Modules registry with authentication and support for multiple backends.
*   [terrareg (‚≠ê295)](https://github.com/MatthewJohn/terrareg) - Open-source modules registry with UI, optional Git integration and deep analysis.
*   [terustry (‚≠ê66)](https://github.com/veepee-oss/terustry) - Proxy registry for providers.

### Helpers

*   [terratag (‚≠ê974)](https://github.com/env0/terratag) - CLI tool allowing for tags or labels to be applied across an entire set of OpenTofu/Terraform files.
